tarasister-analytics プロジェクトの修正と反映を一気に進めてください。

## 作業内容

### 1. コード.js の修正
CONFIG.CONVERSION_EVENTS を以下のみに変更:
['purchase']

理由: GA4のキーイベントは purchase のみ設定されている。
page_view や generate_lead, contact_form_submit は未設定のため、
含めると数値が実態と乖離する。

### 2. appsscript.json を以下の内容に完全置換

{
  "timeZone": "Asia/Tokyo",
  "dependencies": {
    "enabledAdvancedServices": [
      {
        "userSymbol": "AnalyticsData",
        "version": "v1beta",
        "serviceId": "analyticsdata"
      }
    ]
  },
  "webapp": {
    "access": "ANYONE_ANONYMOUS",
    "executeAs": "USER_DEPLOYING"
  },
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8"
}

変更点:
- timeZone を Asia/Tokyo に
- AnalyticsData を advancedServices に明示宣言
- webapp 設定を manifest に追加（再デプロイ時の再設定不要化）

### 3. 反映
cd X:/projects/tarasister-analytics
clasp push

### 4. 確認
clasp versions を実行して結果を報告

### 5. 動作確認手順の案内
push 成功後、私（ユーザー）がブラウザで Apps Script エディタを開いて
testConnection → exportAllReports を手動実行する流れを案内してください。

## 注意事項
- コード.js の他の関数ロジックには触らないこと
- .clasp.json は変更しないこと
- push 前に変更差分を diff 形式で見せてから実行すること
